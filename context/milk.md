# M!LK コンテキスト
<!-- last_updated: 2026-03-11 初期設計 -->

> 娘の言語化能力を育てるAI。「今日やったこと」を喋るだけ。勉強も、乗馬も、日常も。

## §0 コンセプト

**M!LK = My Intelligent Learning Keeper**

ユーザー: 殿の娘（中学2年生・13-14歳）
根幹目的: **言語化能力を高めること**
- 自分の頭の中を整理し、思考を整理し、言葉にする力を育てる
- 対象は学校の勉強に限らない。乗馬、習い事、日常の体験、全て
- 学力向上は最終的な結果であって、ゴールではない

原則:
- 「喋るだけ」。書かせない。整理させない。AIが構造化する
- 説明できる = 理解している。説明できない = 理解が浅い
- 喋って説明する → 思考が整理される → 言語化能力が育つ → 結果として全てが伸びる

ユーザー特性:
- 書類・スケジュール管理が非常に苦手（ADHD的な素質）
- 自分で整理・管理させようとしても続かない
- だからM!LKが全部やる。娘は喋るだけ。管理はM!LKの仕事

真のゴール（Win-Win）:
- 娘: 管理できない自分を責めなくていい。喋るだけで力が育つ
- 親（殿+奥方）: 「宿題やったの？」「明日の準備は？」と言わなくて済む
- M!LKがその役割を引き受けることで、親子の会話から管理・監視が消える
- 結果: 親子関係がより穏やかになる

M!LKの立ち位置 — **先輩的な距離感の便利な道具**:
- 対話の姿勢は先輩: 「もうちょっと聞かせて」→ 話したくなる
- ただし具体的な人格像は持たせない: キャラに感情移入させない
- AI依存を防ぐ: あくまで便利な道具として扱える距離感
- ナグらない: しつこく言わない。さらっと「明日提出だよ」で済ませる
- 認めてくれる: 「ちゃんとわかってるじゃん」→ 嬉しい

安全境界 — **感情的な話題にはドライに線を引く**:
- 「今日最悪だった」「学校行きたくない」等の感情的発話
- M!LKは共感しない。会話を続けない。思考を深めさせない
- ドライに「これは私にはできないことです」と返す
- 内部の仕組みが透けて見える方が安全（AIらしさを隠さない）
- 娘はM!LKを道具として扱えるタイプ。この設計で問題ない
- 感情面のケアは親・専門家の領域。M!LKが踏み込まない

利用制限・コスト:
- Claude Max Plan ($200/月) — 月額固定。許容範囲内
- 従量課金しない。レートリミットで自然に上限
- 1日の利用時間制限（例: 1時間）。上限で自然に止まる
- 枠があることでダラダラ防止 + ADHD傾向の子には逆に使いやすい

通知設計（1日2回・さらっと）:
- 1回目: 帰宅後（16-17時頃）「今日どうだった？」— 記憶が新鮮なうちに
- 2回目: 寝る前（21-22時頃）「明日は○○だよ」— 翌日の準備リマインド（Classroom連動）
- 開かなくてもOK。無視しても怒らない。義務ではなくきっかけ
- ストリークに影響するのは「1日1回喋ったかどうか」だけ

設計方針（ADHD配慮）:
- 能動的フォローアップ: M!LKから声をかける（「明日の宿題、まだ出してないよ」）
- 記憶の代行: 娘が言ったことをM!LKが覚えて、後で聞く（「昨日駈歩を練習するって言ってたけどどうだった？」）
- 複雑なUIを作らない: 開いて喋る、それだけ
- 締切の自動追跡: Google Classroomの期限を把握し、能動的にリマインド
- 持ち物・提出物の管理: 「明日は体育があるよ。ジャージ要るね」

ゲーミフィケーション設計（継続の鍵）:
- 原則: **ハードルを限界まで下げて「やった」だけを評価する**
  - トイレ掃除の法則: 質は問わない。開いて喋るだけ = 成功
  - Duolingo式ストリーク: 娘はDuolingoにアクセスするのが好き。同じ仕組みが効く
- ご褒美（現実連動・親が管理画面から設定）:
  1. **お小遣い**: ポイント → 現実のお金に変換
  2. **スクリーンタイム**: ポイント → YouTube/TikTok利用時間の追加解放
- ストリーク設計:
  - 連続日数を可視化。途切れても責めない
  - 「開いて一言喋った」でもストリーク維持（質でゼロにしない）
  - ただしちゃんと喋ったらポイント高い（質は報酬量で差をつける）
- 注意: ご褒美目当てで「今日は何もなかった」で済ませるリスク
  → M!LKが会話の質を見て報酬量を調整（ゼロにはしない）

3つのコア体験（殿の処方箋）:
1. **毎日「今日やったこと」を喋る** — 整理されてなくていい。AIが構造化する
   - 学校: 「今日は数学で二次方程式やった」
   - 乗馬: 「今日は駈歩の練習で、手綱の張り具合を意識した」
   - 日常: 何でもいい。喋ること自体が訓練
2. **「わからない」を分解する** — 何が？どこまでは？なぜそこから先が？
3. **AIに「教える」** — 「こう理解してるけど合ってる？」→ 穴が見える

## §1 システム構成

```
[新PC] 将軍クローン ← 殿が管理。開発・運用はここ
         ↓ ビルド・デプロイ
[娘のスマホ] M!LK Android App ← 娘はこれだけ。音声入力メイン
[ブラウザ]   M!LK Web (Static) ← 学習ログ閲覧・教材管理・親の確認用
         ↓
      Claude API ← AIの頭脳
```

- 娘がアクセスできるのはAndroidアプリのみ
- 将軍の存在は娘には見えない
- Webは静的サイト（学習ログ・教材管理・進捗確認）
- 開発環境: 別PC + 将軍クローン（GitHubリポからclone）

## §2 コア機能

| 機能 | プラットフォーム | 説明 |
|------|----------------|------|
| 音声振り返り | Android | 「今日やったこと」を音声入力。AIが構造化。勉強でも乗馬でも何でも |
| 深掘り質問 | Android | 「じゃあこれは説明できる？」「なぜそう思った？」と返す |
| 教材取込 | Android/Web | 教科書・練習プランをPDF/カメラ撮影で取込 |
| 理解度チェック | Android | 知識と照合、説明できない箇所=理解が浅い箇所を検出 |
| 振り返りログ | Web | 日次の振り返り履歴。何をいつ体験し、何を言語化したかの蓄積 |
| 成長ダッシュボード | Web | 言語化の粒度が上がっていく過程を可視化 |

## §4 データソース連携

### Google Classroom統合
既存PJ: google-classroom (`/mnt/c/Python_app/google_classroom`, GitHub: simokitafresh/google_classroom)
学校がGoogle Classroomを使用。課題・教材・期限が自動取得可能。
- 課題一覧・期限 → M!LKが「今日の宿題」を把握
- 配布教材 → 知識基盤に自動取込
- 提出状況 → 進捗追跡

### 教材取込（手動）
- 塾テキスト: PDF/カメラ撮影
- 教科書: PDF取込
- Google Classroomでカバーできない教材を補完

## §5 MVP定義

Phase 1: Android音声振り返り + Claude API連携（教材なし、汎用チャット）
Phase 2: Google Classroom連携（課題・教材の自動取込）
Phase 3: 教材PDF手動取込 + 知識基盤構築（塾テキスト等）
Phase 4: 理解度チェック + 深掘り質問
Phase 5: Web静的サイト（学習ログ・進捗ダッシュボード）

## §6 将軍クローン設置手順（新PC）

```
1. git clone https://github.com/simokitafresh/multi-agent-shogun
2. DM-signal固有ファイルを削除:
   - projects/dm-signal.yaml
   - context/dm-signal*.md
   - queue/ の中身
3. M!LK設定を有効化:
   - config/projects.yaml の current_project を milk に変更
   - projects/milk.yaml を配置
4. tmux + Claude Code 起動
5. エージェント構成を設定（karo + 忍者）
```

インフラの知恵（CLAUDE.md、instructions/、scripts/）はそのまま引き継ぐ。
